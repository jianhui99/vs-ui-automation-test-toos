Test Case ID: TC20
Title: LaunchSettings.json Sync from VS Debug Profile
Feature: Debug Profiles (VS)
Priority: P2
Classification: Functional
Client Type: Visual Studio IDE (WinAppDriver)

Test Data:
  Template = "Console App"
  ProjectName = "LaunchProfileApp"
  Location = {WorkDir_Root}/TC20
  ProfileArg = "--env staging"            # sample command-line argument

Expected Results:
  - A debug profile created/edited in the VS Project Properties > Debug UI is written to
    Properties/launchSettings.json
  - The app runs using the profile configuration (e.g. command-line args applied)

Test Steps:
  1. Launch Visual Studio via WinAppDriver.
  2. Create a "Console App" project; wait for the solution to load.
  3. Open Project > Properties > Debug > General > Open debug launch profiles UI.
  4. Create/edit a profile: set command-line arguments to ProfileArg; save.
  5. Read Properties/launchSettings.json from disk; assert it contains the profile and
     the commandLineArgs value matches ProfileArg.
  6. Start Without Debugging using the profile; confirm the app runs with the args applied
     (e.g. output reflects the staging env).
  7. Assert UI changes and the on-disk launchSettings.json stay in sync. DumpUI on failure.
